# BarberApp Documentation

This folder contains the main planning, architecture, progress, and roadmap documents for BarberApp.

BarberApp is a React Native / Expo mobile app for clients and barbers. The app supports barber discovery, services, availability, bookings, messaging, and future AI-powered haircut/style recommendations.

## Documents

- `01-overview.md` - high-level app vision, current status, and core product flow
- `02-frontend.md` - frontend architecture, routing, screens, and completed UI/features
- `03-backend-firebase.md` - Firebase/Auth/Firestore backend structure and future backend plans
- `04-booking-system.md` - booking data model, slot logic, status rules, and future booking work
- `05-messaging-system.md` - conversation/message structure and real-time messaging plan
- `06-ai-roadmap.md` - AI vision analyzer, chatbot, RAG, model, and hosting roadmap
- `07-ui-implementation.md` - UI status, cleanup plans, reusable components, and design polish
- `08-security-rules.md` - Firestore and Storage security rule planning
- `09-future-roadmap.md` - near-term, medium-term, and long-term roadmap
- `progress-log.md` - chronological development progress notes

## Current Development Strategy

The project is being built feature-by-feature and file-by-file to understand the architecture deeply.

Current strategy:

1. Build major app features at a basic functional level.
2. Confirm the data flow works end-to-end.
3. Add security rules and safety checks.
4. Refactor large files into reusable components.
5. Improve UI design and polish.
6. Add advanced features such as notifications, AI, payments, and image uploads.
